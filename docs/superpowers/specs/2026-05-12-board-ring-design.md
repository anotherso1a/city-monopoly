# Board 环形地图设计方案

> 日期：2026-05-12
> 状态：初稿

---

## 1. 方案概述

**新方案（替代原格子方案）：** 不画格子方块，只画一条环形不规则线路，在线上标记点位序号，点击后弹窗显示信息。

### 核心理念
- 线路完全不规则，像真实城市街道
- 点位是路上标记，非独立方块
- 美观简洁，适配任意点位数量

---

## 2. 线路生成算法

### 2.1 点位生成

```
N = 点位总数
angleStep = 2π / N

每个点 i：
  angle = i * angleStep
  radiusVariation = 0.7 ~ 1.3 (随机±30%)
  radius = baseRadius * radiusVariation

  确保相邻点不交叉：
    如果 distance(i, i+1) < minDistance
      调整 radiusVariation 向外
```

### 2.2 路径连接

直线段 + 圆角过渡：

```
每段：lineTo → arcTo(圆角) → lineTo
圆角半径 = 路径宽度的 0.5~1 倍
```

---

## 3. 点位标记

### 3.1 样式

| 元素 | 样式 |
|------|------|
| 点位圆点 | 半径 12px，白底 + 金色边框 |
| 序号文字 | 紧贴圆点旁，垂直朝上 |
| 当前格 | 圆点放大 1.5x，金色高亮 + 发光效果 |
| 非当前格 | 灰底灰边 |

### 3.2 安全距离

- 点位圆点半径 12px
- 点位间最小距离 = 圆点直径 + 8px 间隙
- 半径 ±30% 波动范围内自动保证安全距离

---

## 4. 交互设计

### 4.1 点击检测

```
点击位置 (px, py)
遍历所有点位，找到 distance(px, py) < 20px 的点位
```

### 4.2 高亮效果

- 当前格：金色填充 + 白色边框 + 外圈发光
- 触摸时：该点位轻微放大

### 4.3 弹窗

复用现有 `gridtap` 弹窗逻辑，传递点位索引

---

## 5. 渲染流程

```
render():
  1. 计算所有点位 (x, y)
  2. 清空画布
  3. 保存状态，平移画布到中心
  4. 画路径（lineTo + arcTo 交替）
  5. 画所有点位圆点 + 序号
  6. 当前格高亮
  7. 恢复状态
```

---

## 6. 设计决策汇总

| 问题 | 最终方案 |
|------|----------|
| 线路形状 | 完全不规则环形，随机半径 ±30% |
| 连接方式 | 直线段 + 圆角过渡 |
| 点位样式 | 圆点 + 序号，当前格高亮 |
| 文字方向 | 始终垂直朝上 |
| 安全距离 | 最小 20px 间距，动态调整半径 |
