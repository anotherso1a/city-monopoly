> 版本：v1.0
> 日期：2026-05-12
> 状态：初稿

---

## 1. 项目概述

### 1.1 项目背景

在中国，城市游玩存在千篇一律的问题。传统的 city-walk 缺乏游戏化机制，导致游玩体验单调、缺乏目标感。City Monopoly 旨在通过 AI 生成结合真实地点的大富翁地图，将线下城市探索转变为具有目标感、社交性和趣味性的游戏体验。

### 1.2 项目简介

City Monopoly 是一款运行在微信小程序中的现实版大富翁游戏。用户通过获取定位、选择游玩范围，由 AI 生成结合真实 POI 地点的大富翁地图。玩家通过投骰子在地图上移动，实际前往目标地点进行打卡，最终生成带照片的时间线分享到朋友圈。

### 1.3 核心价值

| 价值点 | 描述 |
|--------|------|
| 游戏化城市探索 | 将 city-walk 转变为有目标、有奖励的游戏体验 |
| 社交分享 | 生成精美的打卡时间线，满足朋友圈分享需求 |
| 低门槛 | 不需要自建服务器，使用公共服务（高德 POI API、大模型 API） |
| 灵活性 | 支持离线游玩、多人各自游玩、随时结算 |

### 1.4 技术选型

- **平台**：微信小程序
- **不依赖自有服务器**，尽可能使用公共服务
- **地图 POI**：高德地图 POI API
- **AI 生成**：大模型 API（通义千问 / GPT / Claude 等，支持 Function Calling / Tool Use）
- **状态存储**：微信小程序 localStorage（限额 5MB）
- **地图分享**：JSON 文件导入/导出，通过微信聊天发送

## 3. 地图创建模块

### 3.1 流程概述

```
定位 → 选择游玩范围 → 问卷式选项 → AI 生成地图 → 预览确认 → 保存
```

### 3.2 定位与范围选择

| 步骤 | 说明 |
|------|------|
| 定位 | 调用微信 `wx.getLocation` 获取用户当前位置 |
| 范围选择 | 提供预设选项：500m / 1km / 1.5km / 商圈/商场 |
| 自定义范围 | 支持用户手动输入范围半径 |

### 3.3 问卷式选项

创建地图时，向用户展示配置选项：

| 选项 | 类型 | 说明 |
|------|------|------|
| 地图名称 | 文本输入 | 默认为"我的探索-[日期]" |
| 格子数量 | 选择 | 预设：15 / 20 / 25 / 30 / 自定义 |
| 是否允许重复打卡 | 布尔 | 是：可重复打卡；否：每格仅可打卡一次 |
| 起始金币 | 数值 | 默认 1000 |
| 每圈奖励金币 | 数值 | 默认 500 |
| 获胜条件 | 选择 | 达到指定金币 / 完成圈数 / 无终点（自由探索） |
| 机会卡偏好 | 多选 | 纯游戏类 / 结合现实类 / 随机混合 |

### 3.4 POI 获取

调用高德地图 POI API 获取选定范围内的地点：

- **餐饮**：奶茶店、咖啡店、餐厅、小吃
- **购物**：便利店、超市、商场
- **景点**：公园、博物馆、标志性建筑
- **公共设施**：地铁站、公交站、厕所

API 调用参数：
- `types`：多品类筛选
- `radius`：用户选择的范围
- `keywords`：辅助筛选

### 3.5 AI 生成地图

使用大模型 API 生成地图 JSON。

**Prompt 模板结构：**

```
你是一个游戏设计师，需要根据以下 POI 数据设计一个大富翁风格的环形地图。

POI 数据：[JSON 格式的 POI 列表，包含名称、类型、地址]

要求：
1. 生成 [N] 个格子的环形地图
2. 机会卡格子占总格子数的 15-20%
3. 同类 POI 尽量分散，避免连续排列
4. 为每个机会卡格子生成随机事件描述
5. 返回 JSON 格式

输出格式：
{
  "name": "地图名称",
  "gridCount": N,
  "grids": [
    {
      "index": 0,
      "type": "poi" | "chance",
      "poi": { "name": "...", "address": "...", "type": "...", "location": {...} } | null,
      "chanceCard": { "description": "...", "goldChange": +/-N, "suggestCheckin": true/false } | null
    }
  ],
  "config": { ... }
}
```

**验证机制：**
- 检查生成的 JSON 格式是否正确
- 检查 POI 是否都在用户选定的范围内
- 检查机会卡描述是否合理

### 3.6 地图预览与保存

- 显示生成的地图预览（环形布局 + 格子信息）
- 用户可确认保存或重新生成
- 保存到 localStorage，包含完整 JSON 数据

## 5. 游玩流程模块

### 5.1 初始态界面

| 元素 | 说明 |
|------|------|
| 全屏背景 | 游戏主题背景图（骰子、棋子、金币元素） |
| 创建地图按钮 | 中央大按钮，点击进入创建流程 |
| 历史地图入口 | 侧边栏或底部入口，查看已保存的地图 |

### 5.2 游玩态界面布局

```
┌─────────────────────────────────────┐
│ [≡]                           [?]
│                                     │
│         ┌─────────────┐             │
│        │   全览地图   │             │
│        │  （环形格子）│             │
│        │  ★ 当前位置  │             │
│         └─────────────┘             │
│                                     │

│                                     │
│         [📷 拍照打卡]               │
└─────────────────────────────────────┘
```

### 5.3 投骰子流程

1. 用户点击"投掷骰子"按钮
2. 显示骰子动画（1-6 点随机）
3. 动画结束后，玩家的 `currentGridIndex` 向前移动对应步数
4. 如果跨越起点（环形），`currentLap++`，发放每圈奖励金币
5. 显示到达格子的信息（如果 POI 类型，显示地点名称）
6. 如果是机会卡格子，自动抽取一张机会卡并展示

### 5.4 地图交互

| 交互 | 功能 |
|------|------|
| 点击格子 | 查看格子详情（名称、地址、图片、导航、打卡按钮） |
| 捏合缩放 | 放大/缩小地图 |
| 拖动 | 平移地图查看其他区域 |
| 全览模式 | 一屏显示完整环形地图 |
| 当前位置模式 | 以当前格子为中心，显示前后若干格子 |

### 5.5 格子详情弹窗

点击地图上的格子后，弹出详情弹窗：

```
┌─────────────────────────────────────┐
│         奶茶店B                      │
│  ─────────────────────────────────  │
│  📍 朝阳区建国路88号                  │

│                                     │
│  [实景图片]                          │
│                                     │
│  [🧭 导航]  [📷 拍照打卡]            │
│  [❌ 关闭]                           │
└─────────────────────────────────────┘
```

- **导航按钮**：拉起微信 `wx.openLocation` 或调用高德/百度地图 APP
- **拍照打卡按钮**：仅在玩家当前所在格子显示
- **实景图**：从高德 POI API 获取，若无则显示兜底图片

### 5.6 拍照打卡

1. 用户点击"拍照打卡"按钮
2. 调用微信 `wx.chooseImage` 选择或拍摄照片
3. 照片保存到本地，记录打卡信息（格子索引、时间戳、照片路径）
4. 打卡成功后，更新游戏实例状态

## 6. 机会卡系统

### 6.1 机会卡预设模板

AI 生成地图时，预设以下类型的卡牌：

| 类型 | 金币变化 | 描述模板 |
|------|----------|----------|
| 纯奖励 | +100 ~ +500 | "恭喜获得 XXX 金币！" |
| 纯惩罚 | -100 ~ -500 | "丢失 XXX 金币..." |
| 饮品类 | -150 ~ -300 | "你太渴了，购买饮品花费 XXX 金币..." |
| 餐饮类 | -200 ~ -500 | "肚子饿了，找家餐厅饱餐一顿花费 XXX..." |
| 交通类 | -50 ~ -150 | "需要打车，代步花费 XXX 金币..." |
| 意外收获 | +200 ~ +800 | "捡到 XXX 金币！" |
| 好运降临 | +500 ~ +1000 | "运气爆棚，获得 XXX 金币！" |

### 6.2 机会卡抽取逻辑

- 每个机会卡格子预设 3-5 张候选卡牌
- 玩家到达时，前端随机抽取一张
- 抽取后从候选池移除，保证每个格子的每张卡只抽一次
- 所有卡牌在游戏开始时已生成，无需网络

### 6.3 用户自定义机会卡

- 在创建地图时，可添加自定义机会卡
- 可删除 AI 生成的不喜欢的卡牌
- 自定义卡牌格式：`{ description, goldChange, suggestCheckin, checkinType }`

### 7.3 结算展示

结算时显示金币余额，以及金币变化明细。

## 8. 地图分享与存储

### 8.1 导出地图

1. 用户点击"分享地图"
2. 将地图 JSON 序列化为字符串
3. 生成 `.json` 文件
4. 调用微信 `wx.shareFileMessage` 或通过聊天发送文件

### 8.2 导入地图

1. 用户在微信聊天中打开 `.json` 文件
2. 小程序通过关联的文件类型打开
3. 解析 JSON，验证格式
4. 保存到 localStorage

### 8.3 存储容量

- localStorage 上限约 5MB
- 每个地图 JSON 约 5-20KB
- 预计可存储 50-100 个地图
- 超出时提示用户清理旧地图

### 9.1 结算触发

- 用户主动点击"提前结束"

- 达到获胜条件（若设置）

### 9.3 距离计算

- 每次玩家导航到新格子时，通过高德/百度路线规划 API 获取预估距离
- 累加所有导航距离作为总徒步距离
- 若无导航记录，按平均每格距离估算（根据地图范围和格子数量计算）

### 9.4 生成分享图

- 尺寸：竖版 2:3 比例，适合朋友圈
- 内容：照片时间线 + 统计数据 + 小程序码
- 底部嵌入小程序码，引导新用户扫码体验

## 11. 地图编辑功能

### 11.1 编辑模式

用户可手动编辑 AI 生成的地图：

| 操作 | 说明 |
|------|------|
| 新增格子 | 在环形路线上插入新格子 |
| 删除格子 | 移除不需要的格子，重新编号 |
| 修改格子 | 编辑 POI 信息或机会卡内容 |
