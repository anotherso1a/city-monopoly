# City Monopoly UI 设计提示词

> 日期：2026-05-29
> 版本：v2.0
> 类型：移动端小程序 UI 设计规范
> 风格：手绘插画风格（Hand-drawn Illustration）
> 目标：一次性生成所有子页面的高保真 UI 草图，使用统一色彩与组件规范

---

## 全局设计规范

### 设计理念

**City Monopoly（城市大富翁）** 是一款基于真实城市地图的步行打卡游戏。核心玩法：玩家通过投骰子在城市环形棋盘上移动，完成打卡任务收集金币，最终结算时根据打卡数量和收集金币计算成绩。

**设计风格**：现代、亲和、城市探索场景化；浅色为主，支持深色模式。

**手绘插画风格核心特征**：
- 线条：不规则、有轻微抖动的手绘线条，模拟铅笔/钢笔绘制效果
- 形状：圆润的边角，不完美的几何形状，有机的曲线
- 质感：轻微的纸张纹理背景，留白感强，不拥挤
- 色彩：柔和但饱和度适中，平涂为主，无过度渐变
- 阴影：柔和的投影，轻微的物体立体感

**视觉参考**：城市旅行手账插画、文创品牌插画风格（如故宫文创）、轻量游戏界面设计（如动物森友会）

### 排版规范

```
字体选择：
- 标题/Logo：圆体字（临时用系统黑体）
- 正文：系统默认字体，保持清晰可读
- 数字/金币：使用等宽字体

字号梯度（rpx）：
- 大标题（H1）：48rpx
- 页面标题（H2）：36rpx
- 模块标题（H3）：32rpx
- 正文：28rpx
- 辅助文字：24rpx
- 标签/角标：20rpx

行高：标题 1.2，正文 1.5，辅助 1.4
```

### 主色调与强调色

```
主色（财富金）：#FFB800
- 用于：主按钮、Logo、高亮、进度指示
- 象征：财富与成功

强调色：
- 机会卡：#C41E3A（深红渐变）
- 成功/打卡：#4CAF50（幸运绿）
- 天空蓝：#29B6F6（链接、水面）
- 樱花粉：#F48FB1（装饰、提示）

中性色：
- 背景：#F5F9FF（天空白，带蓝调的米白）
- 卡片：#FFFFFF
- 文字主色：#2D3436
- 文字次要：#636E72
- 分割线：#E0E0E0
```

### 组件系统

**通用组件**：顶部导航栏、底部操作区、卡片、标签、步骤指示器、空状态插画、骨架屏、Toast 提示、Bottom Sheet、侧边栏抽屉、弹窗 Modal。

**图标风格**：线性风格 + 手绘轮廓，食材/地点类图标有细微手绘感。

**交互规范**：
- 所有可点击控件 ≥ 44pt（大拇指区域）
- 按钮按下态：scale(0.96) + 颜色加深，150ms
- 卡片/按钮圆角：12–24rpx（非标准值，模拟手绘感）
- 阴影柔和，分割线尽量少，用留白分区

**无障碍**：对比度 ≥ WCAG AA；语音读屏可识别控件语义。

### C3. 环形棋盘设计规范

```
线路生成算法：
- 点位数量：N 个，均匀分布在圆周上
- 半径波动：0.7 ~ 1.3（随机±30%），形成不规则感
- 连接方式：直线段 + 圆角过渡（圆角半径 = 路径宽度 0.5~1 倍）

点位样式：
- 点位圆点：半径 24rpx，白底 + 金色边框 2rpx
- 序号文字：紧贴圆点旁，垂直朝上
- 当前格：圆点放大 1.5x，金色填充 + 外圈发光
- 已打卡格：绿角标 #4CAF50

手绘风格处理：
- Canvas 绘制时线条使用轻微抖动效果
- 连接线略有不规则感
- 点位使用手绘风格圆点效果

棋盘背景：天空蓝渐变 #E3F2FD → #BBDEFB
```

## J. 手绘风格实现技巧

### J2. 手绘风格阴影

```css

  box-shadow:
    0 8rpx 24rpx rgba(0,0,0,0.06),
    0 2rpx 8rpx rgba(0,0,0,0.04);

    0 4rpx 12rpx rgba(0,0,0,0.08);

```

### J3. Canvas 手绘线条

```javascript
// 模拟手绘线条抖动效果
function drawHandDrawnLine(ctx, x1, y1, x2, y2) {
  ctx.beginPath();
  ctx.moveTo(x1, y1);

  // 在两点之间添加随机抖动
  const segments = 5;
  for (let i = 1; i <= segments; i++) {
    const t = i / segments;
    const jitter = (Math.random() - 0.5) * 4; // 抖动幅度
    const x = x1 + (x2 - x1) * t + jitter;
    const y = y1 + (y2 - y1) * t + jitter;
    ctx.lineTo(x, y);
  }

  ctx.stroke();
}
```
